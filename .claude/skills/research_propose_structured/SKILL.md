# research_propose_structured

## 目的

調査・提案を「構造化された結論」で返す。実装と混ざらないようにする。

## 入力

- 調査テーマ
- 制約（コスト・工数・技術スタック等）
- 優先度（速さ / 安全性 / 可逆性 / その他）
- 欲しいアウトプット（選択肢比較 / 推奨案 / Next Action等）

## 出力

**1. 結論**（最初に、1〜3行）

**2. 選択肢比較**

| 選択肢 | 利点 | 欠点 | 前提条件 |
|------|-----|-----|--------|

**3. 推奨案と理由**（2〜4行）

**4. 具体Next Action**（gh issue 化できる粒度）

```
- [ ] <action A>
- [ ] <action B>
```

**5. リスク / 不確実性**

```
- <リスクX>: <影響> / <緩和策>
- <不確実な点Y>: 確認できていないため推定
```

**6. 参照根拠**

```
- <ファイル or URL>: <参照した理由>
```

## 制約

- 実装しない
- コードを書かない（必要な場合は疑似コードまで）
- 参照した根拠（リンクやファイル）を必ず列挙する
- 「やるべき」という断定を避ける — 選択肢を整理するにとどめる

---

**呼び出し例:**
> "Research options for adding a new adapter. Load `.claude/skills/research_propose_structured/SKILL.md` and follow it."
