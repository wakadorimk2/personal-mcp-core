# implement_issue_minimal

## 目的

1 issue = 1 skill 呼び出し。実装→最小テスト→コミット単位の作業をスコープ内で完結させる。

## 入力

- issue本文（Goal / Scope / 完了条件）
- 現在のリポジトリ状況（ブランチ、関連ファイル）

## 出力

**1. 実装方針**（箇条書き、3〜5行）

```
- <変更の核心を一言で>
- <影響範囲>
- <採用しなかった選択肢があれば一行で>
```

**2. 変更ファイル一覧**

| ファイル | 変更種別（新規/編集/削除） | 変更理由 |
|--------|------------------------|--------|

**3. 実行コマンド**

```bash
# 例
pytest tests/path/to/test.py -v
diff AI_GUIDE.md src/personal_mcp/AI_GUIDE.md
```

**4. 完了チェックリスト**

- [ ] Scope内の変更のみ
- [ ] 仮定欄の内容を人間が確認済み（破壊的変更はない）
- [ ] テストが通る
- [ ] コミットメッセージが規約に沿っている

## 制約

- 仕様の追加提案をしない
- Scope外を勝手に実装しない
- 「足りない情報」は質問ではなく **仮定** 欄に明記して進める（ただし破壊的変更は止める）
- リファクタや整理は issue に書かれていなければしない

## 仮定欄テンプレート

```
## 仮定
- <前提A>（確認できていないが破壊的でないため続行）
- <前提B>（issue に記載なし／既存実装から推定）
```

---

**呼び出し例:**
> "Implement GitHub issue #25. Load `.claude/skills/implement_issue_minimal/SKILL.md` and follow it."
