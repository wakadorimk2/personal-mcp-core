# personal-mcp-core
個人的な前提や原則をAPIとして提供するMCPサーバーです。

このリポジトリは、個人用の MCP / コンテキストサーバーの中核です。

AIを賢くするためのものではなく、
AIが私を誤解しにくくするための「前提」と「原則」を、
構造として外に出すことを目的としています。

会話の中で積み上がる記憶ではなく、
変わりにくい思考の地面（ground）を用意する。
それがこのリポジトリの役割です。

## これは何か

- Python ベースで育てていく個人用リポジトリ
- API / MCP 経由で LLM やツールに接続される前提の「文脈提供装置」
- 知識ではなく、判断の前提・優先順位・態度を扱う場所
- モデルやサービスが変わっても、同じ前提から思考を始めるための基盤

## これは何ではないか

- 万能な AI フレームワークではありません
- 生産性ハック集でもありません
- 判断や意思決定を AI に委ねるためのものではありません

AI は「代わりに考える存在」ではなく、
「誤解せずに補助する存在」として扱います。

## 背景にある考え方

私は、AI が過去の会話を「覚えている」ことに安心感を感じてきました。
しかしその安心の正体は、記憶そのものではなく、

- 毎回ゼロから説明しなくていいこと
- 前提が共有されている状態から思考を始められること

だと気づきました。

このリポジトリは、
「覚えていてほしいこと」を期待する代わりに、
「誤解されたくない前提」を固定するためのものです。

記憶ではなく、構造。
会話ではなく、プロトコル。

## コアアイデア

- 思想や価値観は、プロンプトではなく構造に落とす
- 判断を AI にさせない代わりに、判断の土台を渡す
- 強いときの自分が、弱っているときの自分を追い詰めない設計にする

最適解よりも納得感を、
速さよりも持続性を、
正しさよりも壊れにくさを優先します。

## 構成（予定）

- `AI_GUIDE.md`  
  AI がこのリポジトリを使うときに守るべき態度・優先順位・禁止事項を、
  人間が読める言葉で記述します。

- `src/`  
  上記の前提や参照情報を、API / MCP 互換の形で提供する Python 実装。
  初期段階では最小構成に留めます。

- `data/` や `notes/`（必要になったら）  
  名著の要約や参照テキストなど、思想を補助する材料。
  思想そのものとはレイヤーを分けて管理します。

## ステータス

このリポジトリは、意図的に未完成です。

一気に作り切るものではなく、
考えが変わったときに書き換え、
疲れたときに立ち戻れる場所として、
ゆっくり育てていきます。

思考の盆栽のように。
