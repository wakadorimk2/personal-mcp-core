# personal-mcp-core

個人の活動ログ（エンジニアリング・イラスト・ゲームなど）を
時間軸で蓄積・検索・集計するための基盤です。

これはチャットボットでも、性格診断でも、特定ゲーム専用ツールでもありません。
行動を後から意味づけできる形で保存するための土台です。

現在は CLI ベースで動作します。
将来的に UI や外部接続を追加する可能性がありますが、コアは常にシンプルに保ちます。

## これは何か

* 個人の活動ログを構造化して保存する基盤
* 後から分析・可視化・解釈できるようにするためのイベントストア
* 解釈レイヤー（LLM / 統計 / ルール）を差し替え可能にする設計

このリポジトリの目的は「賢く解釈すること」ではなく、
**意味を後付け可能なログ構造を保つこと**です。

## 何ができるか（現在）

```bash
# 活動ログを追加する
python -m personal_mcp.server log-add "アトラスの外縁でボス撃破" \
  --domain=game --kind=milestone --tags=atlas,boss

# 直近のログを一覧する
python -m personal_mcp.server log-list --n 10

# ドメイン・タグ・期間で絞り込む
python -m personal_mcp.server log-list --domain=game --since 2026-03-01
```

ログは `data/logs.jsonl` に追記されます。

**注意**: 個人情報や機密情報を含めないでください。

## ドメインの考え方

この基盤は特定用途に固定されません。

例:

* `game` — PoE2 などのプレイログ
* `art` — イラスト制作ログ
* `engineering` — 開発作業ログ
* `life` — 体調・気分など

ドメインは追加可能です。

## 設計思想

### 1. Model-Agnostic

LLM は「解釈器のひとつ」に過ぎません。
この基盤は特定のモデルに依存しません。

### 2. Temporal-First

すべてのログは時間軸を中心に扱います。
重要なのはラベルではなく「変化」です。

### 3. Interpretation is Pluggable

* ルールベース集計
* 統計分析
* LLM サマリ生成

は後から追加できます。

### 4. Core is Small

UI や外部接続は拡張。
コアはログ構造と検索性に集中します。

## 構成

* `AI_GUIDE.md` — AI が守るべき態度・優先順位・禁止事項
* `CLAUDE.md` — Claude Code 向けの作業ガイド
* `src/personal_mcp/` — ログの記録・検索・アダプタ層
* `contracts/` — イベントスキーマ定義（予定）
* `docs/` — 設計メモ・将来構想
* `data/` — ログデータ

## どこへ向かうか

1. **いま** — CLI でログを記録・検索できる
2. **次** — Web UI による一覧・入力（Habitify 置き換え）
3. **その次** — 週次サマリ・傾向可視化
4. **将来** — 必要なら外部連携（MCP / API / Widget など）
