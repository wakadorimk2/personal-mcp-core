# Issue #213: 日常ログ入力の操作摩擦メモ

`#213` の受け入れ条件に合わせ、現行 dashboard 主導線で日常利用時に目立つ摩擦を整理する。

## 今回実装した改善

1. 保存 CTA を現在の入力内容に追従させる

- 候補タップ後や入力中に、保存ボタンを `「...」を保存` に切り替える。
- 入力が空の間は保存ボタンを disabled にし、次アクションを明確にする。

2. 入力コンポーザを画面下部に sticky 表示する

- 候補一覧やサマリー一覧を見たあとでも、入力欄と保存ボタンを同じ位置に維持する。
- `safe-area-inset-bottom` を加えてモバイル下端でも押しやすさを落とさない。

3. キーボード利用時の完了フローを短縮する

- textarea に `enterkeyhint="done"` を付与し、`Enter` 単押しで保存できるようにする。
- 保存成功時は入力欄を blur して、完了メッセージを見やすくする。

## 追加の follow-up 候補

1. fixed candidate の一部をワンタップ即時保存にする

- cold start 用の固定候補のうち、編集前提が低いものだけ即時保存チップ化する。

2. 保存完了フィードバックを composer 近傍の toast に寄せる

- いまは本文テキストで通知しているため、視認性を少し上げる余地がある。

3. 候補チップに直近利用の視覚フィードバックを追加する

- 最後に使った候補を短時間だけ強調すると、連続入力時の迷いを減らせる。
